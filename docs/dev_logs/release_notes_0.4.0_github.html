🦙 LlamaBot v0.4.0 - Scheduled Jobs, Git UI Integration, and New Agent Modes

Release Date: February 22, 2026

🚀 What's New

📅 Scheduled Jobs System ⭐

Automate agent tasks with cron-based scheduling directly from the UI.

  • Cron-based automation - Schedule agents to run on any cron schedule
  • Full UI support - Create, view, and manage scheduled jobs from the interface
  • Easy installation - Install scheduler via /install-scheduler-cron slash command
  • Comprehensive logging - Background job execution with detailed logs

🤖 New Agent Modes ⭐

Specialized agents for database operations and research tasks.

  • Database Mode Agent - Direct ActiveRecord operations from chat using rails_user_mode_agent
  • Delegate Research Agent - Read-only sub-agent for investigation tasks without modifying code
  • User Feedback Agent v2 - Enhanced feedback mode that can now write feedback directly

🔗 Git & GitHub Integration (UI) ⭐

Manage Git directly from the Leonardo interface without needing VSCode.

  • Git checkpoints in UI - View commit history and save/discard changes directly
  • GitHub authentication - /gh auth login with automatic code copy to clipboard
  • SSH setup command - Streamlined developer onboarding with /ssh-setup
  • Push to GitHub - New cloud push icon for pushing checkpoints to GitHub
  • Git config wrapper - /gh commands for git configuration

👤 Per-User Configuration

Fine-grained control over agent visibility by user role.

  • visible_agents setting - Configure which agents each user can access
  • Role-based defaults - Different default agents based on user role
  • Engineering role enforcement - Slash commands restricted to engineer role

🔧 Improvements

LLM & Model Updates

  • Gemini 3.1 Pro default - Upgraded default Gemini model to 3.1 Pro
  • Simplified Gemini setup - Now activates with just GEMINI_API_KEY (no longer requires GOOGLE_API_KEY)

Browser UX

  • Tab notification badges - Visual indicator when Leonardo finishes tasks (success or error)
  • Dynamic favicon - Shows running, completed, or error states
  • Smart refresh - Refresh button updates all iFrames (except VSCode)

Ticket Mode

  • Light research phase - Performs preliminary research before grounding user observations/stories

System Reliability

  • Bash output truncation - Prevents context overflow with intelligent truncation
  • Critical error detection - Bash tool now detects and highlights critical errors
  • Checkpoint storage migration - LangGraph checkpoints now save to LlamaBot DB instead of LlamaPress
  • Automatic checkpoint cleanup - Protects disk space and DB size on Leonardo instances
  • S3 database restore - New /restore_s3.sh command for database backup restoration
  • Auto SSH key copy - Automatically copies SSH keys from host to VSCode container

Slash Commands

  • /restart improvements - Now includes --follow flag for easy log tailing
  • /gh wrapper - Git config and GitHub CLI operations

🐛 Bug Fixes

  • Fixed Ubuntu permission issues from tool calls
  • Fixed bash output truncation edge cases

⚠️ Migration Notes

  • Checkpointer data location - Now stored in LlamaBot database (automatically migrated)
  • Slash command permissions - Users must have engineer role to execute slash commands
